Github博客同步到Coding,自定义域名双线解析

七月 22, 2019 发布在 教程, Github

　 　　为了解决Github屏蔽了百度爬虫导致的Github Pages无法被收录的问题，将Github上的静态网页同步到Coding Pages上。同时对域名采用DNSPod实现域名双线解析，以加快国内的访问，并被百度收录。这里，以我使用Hugo框架搭建的blog为例，使用在Godaddy上购买的域名，作为自定义域名。

准备工作

  • 已经使用hugo在Github上搭建了个人博客
  • Coding Pages仓库的建立（用于同步Gibhub的静态文件）

  • DNSPod账号注册（用于自定义域名的双线解析，域名服务器支持双线的请忽略）

    Coding Pages仓库的建立

    　　前往Coding注册个人版账号(现已升级为腾讯云开发者平台，支持微信和qq登陆)。此处略过注册过程，如果采用微信或者QQ快捷登陆，建议在个人设置中修改默认生成的用户名。
    　　建立一个新建项目，如下图所示：

    　　填写必要的项目信息，项目命名最好自己好记容易识别（建议勾选read.me初始化项目）：
    　　选择左侧边栏的的Pages服务，在阅读并勾选了同意服务条款后创建Pages： 　　至此，Coding Pages服务的仓库就已经建立完成，接下来需要在git中添加第二个远程仓库的url，并且实现同步提交至远程库。

    Git中添加第二个远程仓库

    　　以下git的操作请在hugo生成的静态页面输出目录下执行，一般是博客目录下的public文件夹。
    　　由于已经链接过Github的仓库，这里只需要再添加一个新的远程仓库即可，代码如下：

    git remote set-url --add origin XXXXXXXXXXXXX

    　　这个命令中的origin是可以更改的，但是为push命令一条便可以同时完成向两个远程仓库的推送，我们就采用和Github上相同的仓库标识。
    　　”XXXXXXXXXXXXX“是需要添加的远程仓库的名字，也就是我们要添加的Coding的仓库地址，如下所示：
    　　在Coding Pages仓库中左侧菜单中找到代码/代码浏览菜单，进入后将右上角的链接复制代替命令中的XXXXXXXXXXXX占位符。
    　　然后我们需要验证一下是添加成功，使用以下命令查看git远程仓库有哪些：

    git remote -v

    　　以我自己的为例，可以看到有Github和Coding两个都叫做origin的远程仓库：

    origin  https://github.com/SAquarius/SAquarius.github.io.git (push)
origin  https://git.dev.tencent.com/SAquarius/SAquarius.git (push)

    把站点文件推向两个远程仓库

    　　由于pull命令只能拉取第一个远程仓库，不能拉取第二个远程仓库，可以采用修改两个仓库的顺序然后拉取第二个仓库，不然第二个仓库无法成功上传。其实也可以采用不同的名字 然后分别拉取和推送。
    　　为了省事，统一命名为origin的情况下，第一次直接强推上去，毕竟Coding上只有一个初始化的readme.md文档，强制推上去它就没有了，不会有什么影响。其命令如下：

    git pull origin master
git push origin master -f

    　　第一次push时，新的Coding仓库需要输入账户和密码，之后就不用了，往后的提交方式和正常一样，不需要添加“-f”的指令了。

    域名DNS服务器设置

    　　在购买的域名中设置DNS服务交给DNSPod管理，具体设置方法如下：

    //将域名DNS服务器改为DNSPod的服务器，不再使用域名提供商Godadyy提供的DNS服务
//以下为DNSPod的服务器映射地址
f1g1ns1.dnspod.net
f1g1ns2.dnspod.net

    　　在DNSPod中加入一条CNAME的记录，指向“你的Pages名字.coding.me”，示例如图所示：
    　　此时，先不要添加Github Pages的CNAME记录，你需要回到Coding Pages的设置中申请SSL安全证书，稍等片刻，一般证书便能申请到。

    SSL安全证书需要每三个月重新申请一次，再次申请的时候，先停用Github的海外解析线路再申请，否则，会因为域名服务器在国外导致申请失败

    双线解析测试

    　　分别测试使用不同的方式访问博客：

    1.指向Coding的测试，使用如下访问
https：//你的pages名称.coding.me   
2.指向Github的测试，使用如下访问
https：//你的仓库名称.github.io

    　　如果都能成功访问到你的博客站点，证明双线解析生效，之后就可以愉快地同时在Coding和Github上更新博客了。

    友情提示

    　　如果像我一样在博客中添加了valine的评论系统，需要在Leancloud的安全中心中的Web安全域名中加入Coding Pages的访问地址。
